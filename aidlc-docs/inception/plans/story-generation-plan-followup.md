## FQ1. ペルソナ数とソフト/ハード差の表現方法 (最重要)

**問い**: Q1 で「2 ペルソナ」、Q3 で「ソフト推し / ハード推し / ソフト推され / ハード推され」とあります。どの構造で生成しますか？

- **A**: **ペルソナを 4 に拡張** — 推し (ハード) / 推し (ソフト) / 推され (ハード) / 推され (ソフト)。各ペルソナにストーリーを 1 本ずつ + 共通 2 = 計 6 ストーリー (Q1 を A → B 相当に変更)
- **B**: **ペルソナは 2 のまま、ストーリー側でソフト/ハード を分割** — ペルソナ「推し側代表」は経済状況・課金強度のレンジ (例: 月 0〜10 万円幅) を内包する形で書き、ストーリー US-OSHI-01 (ハード推しシナリオ) と US-OSHI-02 (ソフト推しシナリオ) で具体例を示す
- **C**: **ペルソナは 2、ストーリーでもソフト/ハード を統合** — 1 本のストーリー内に「ハード推しの場合 / ソフト推しの場合」両方の受入基準を併記。推し / 推され で各 1 本 + 共通 2 = 計 4 ストーリー (Q3 を 6 → 4 に削減)
- **D**: その他 (自由記述)

**AI 推奨**: **A (ペルソナを 4 に拡張)**。理由:
- 「ハード推し: 経済余裕ありズブズブ支援」「ソフト推し: ちょっと余裕でほどほど」は **経済状況・課金強度・利用頻度・心理動機が大きく違う** ため、同一ペルソナに収めると Q2 で追加した MBTI / SNS 利用動向の設定値も矛盾する (1 ペルソナに MBTI が複数入らない)
- ペルソナを 4 にしても、ストーリー数は 6 のままで増えないため、審査向け可読性 (「数を減らしたい」というユーザー意図) に反しない
- 各ペルソナ → 1 ストーリー の対応がきれいに揃い、デモ動線 (Q9) との紐付けも明快
- MVP デモ動線では「ハード推し × ハード推され」のペアを使い、ソフト側は審査向けの "幅の表現" として残せる

**B 案の懸念**: ペルソナの説得力が下がる (「経済余裕 0〜10 万円幅」のペルソナは具体性がなく審査向けに弱い)
**C 案の懸念**: ストーリー 4 本まで減らすと、ソフト/ハード の差別化が逆に見えにくくなる

[Answer]: A (ペルソナを 4 に拡張)

## FQ2. 推され側ソフト / ハードの区分軸の確定

**問い**: 推し側のソフト/ハードは「経済的余裕の量 = 課金強度の差」と明示されました。推され側のソフト / ハードはどう区分しますか？

- **A**: **AI 依存度の差** — ハード推され = 感情労働 100% AI 代行 (本人は承認のみ) / ソフト推され = 部分代行 (本人も一部返信)
- **B**: **収入依存度の差** — ハード推され = 推され収入が主収入 (働かないでも生活できる) / ソフト推され = 副収入・お小遣い感覚
- **C**: **A + B の複合** — ハード推され = 主収入 + AI 依存度高 / ソフト推され = 副収入 + AI 依存度低 (一貫性のある対比)
- **D**: **承認欲求 / 自己肯定感の差** — ハード推され = 承認爆撃を必要とする (重度) / ソフト推され = 日常の小さな褒めで満足
- **E**: その他 (自由記述)

**AI 推奨**: **C (主収入×AI 依存度高 vs 副収入×AI 依存度低)**。理由:
- 推し側の区分軸 (経済余裕の量) と対称になる (主収入か副収入かは経済軸)
- 同時に「AI に任せる」という本サービスの核を表現できる (AI 依存度の差)
- 5/4 メモ「直球で言うと『ダメ人間なので働きたくない人』」(=主収入ハード型) と 5/6 メモ「忙しくて感情労働を外注したい人」(=副収入ソフト型) の両方をカバー

[Answer]: C (主収入×AI 依存度高 vs 副収入×AI 依存度低)

## FQ3. Q2 ペルソナ属性の追加範囲の確定

**問い**: 「B + MBTI と SNS 利用動向を追加」は次のどちらの解釈ですか？

- **A**: **B (標準) + MBTI + SNS 利用動向 のみ追加** — C 案のうち「過去の課金履歴想定 / 推し活経験」は除外
- **B**: **C (詳細) フル採用** — MBTI + SNS + 過去の課金履歴想定 + 推し活経験 すべて含める
- **C**: その他 (自由記述)

**AI 推奨**: **A (B + MBTI + SNS のみ)**。理由: ユーザーは「B + 〜を追加」と書いており、明示された 2 項目以外は標準 (B) のままとするのが素直な解釈。ハッカソン提出物の枚数面でも軽量に済む。

[Answer]: A (B + MBTI + SNS のみ。課金履歴 / 推し活経験は除外)

## FQ4. Q12 のレビュー運用詳細の確定

**問い**: 「C ４人全員でレビューを行います」は次のどちらの解釈ですか？(選択肢 C 自体は「デモ動線対応表だけ全員確認」だったため意図確認)

- **A**: **担当分けなし・全員全件レビュー** — 4 名が `personas.md` / `stories.md` 全文をそれぞれ読んで、修正点を集約
- **B**: **担当分けあり・全員で承認** — 推し担当 / 推され担当 / 共通担当 / 倫理担当 で分担しつつ、最終承認は全員揃った場で
- **C**: **動線表のみ全員確認 + 残りは AI 任せ** — 選択肢 C の文面通り、レビューは動線対応表に絞る
- **D**: その他 (自由記述)

**AI 推奨**: **A (担当分けなし・全員全件)**。理由: ユーザーの「４人全員でレビュー」という表現は分担を否定しているように読める。ストーリー数が 6 + ロードマップ 1〜2 = 計 7〜8 本と少ないため、全員全件でも負担は軽い。

[Answer]: A (担当分けなし・全員全件レビュー)
